# HermesAgent Faceplate — Installers

One-line bootstrap scripts for end users. The packaged Electron app handles routine updates itself via electron-updater; these scripts are the **first-install** + **disaster-recovery** path.

## Usage

| Platform | Command |
|---|---|
| macOS, Linux | `curl -fsSL https://get.hermesagent.app/install.sh \| sh` |
| Windows | `irm https://get.hermesagent.app/install.ps1 \| iex` |

Inspect-first (recommended for first-time installs):

```sh
curl -fsSL https://get.hermesagent.app/install.sh -o hermes-install.sh
less hermes-install.sh
sh hermes-install.sh
```

## Install prefix

Everything lives under one directory so uninstall is a single `rm -rf`:

| Platform | Prefix |
|---|---|
| macOS / Linux | `~/.hermes-app/` |
| Windows | `%LOCALAPPDATA%\Hermes\` |

Override via the `HERMES_PREFIX` env var.

## Release manifest

The script reads a JSON manifest at `https://get.hermesagent.app/channels/stable.json`:

```json
{
  "version": "1.2.3",
  "assets": {
    "macos_x64":   { "url": "https://…/HermesAgent-1.2.3-mac-x64.dmg",  "sha256": "…" },
    "macos_arm64": { "url": "https://…/HermesAgent-1.2.3-mac-arm64.dmg","sha256": "…" },
    "linux_x64":   { "url": "https://…/HermesAgent-1.2.3-linux-x64.tar.gz", "sha256": "…" },
    "linux_arm64": { "url": "https://…/HermesAgent-1.2.3-linux-arm64.tar.gz","sha256": "…" },
    "windows_x64": { "url": "https://…/HermesAgent-1.2.3-win-x64.zip", "sha256": "…" }
  }
}
```

The manifest is generated by CI alongside each release and uploaded to the same CDN/origin as the artifacts. See `docs/v1/research/phase5-installer.md` for the full architecture rationale.

## What still needs to ship before these go live

- **CDN / static origin** for `get.hermesagent.app` (CloudFront / Bunny / Pages — anywhere with HTTPS + ranged GETs).
- **Electron-builder release pipeline** that produces `.dmg` / `.AppImage` / `.tar.gz` / `.zip` and uploads each release to GitHub Releases (or the chosen origin) PLUS writes the manifest.
- **macOS Developer ID cert** + `@electron/notarize` integration — without this, Gatekeeper blocks first launch.
- **Azure Trusted Signing** for Windows — without this, SmartScreen blocks downloaded `.exe`s.
- **Sibling `uninstall.sh` / `uninstall.ps1`** — left out of v1 (`rm -rf $HERMES_PREFIX` does the job manually for now).

These are all packaging + signing work that don't affect the app itself; the installer scripts are written against the manifest shape we'll publish.
